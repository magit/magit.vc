Notes

We are aware that a workflow based on git-send-mail and git-am is more powerful. It is also harder to learn. There was no decision to use Github almost exclusively for these tasks — it has just evolved that way. It appears this is what most contributors prefer.

While we would like to support a workflow native to Git, doing so in parallel to the use of Github is not an option at the moment. Things like duplicated issues and discussions would increase the workload for us maintainers to much. This might change at some point but not now.

We are also considering moving the mailing list to a mailman host and using it to discuss big changes. But as of right now you have to follow the issue tracker on Github to be up-to-date with the latest development.
